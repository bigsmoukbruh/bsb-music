**BSB Music — это специализированное программное решение на базе платформы .NET, предназначенное для потокового воспроизведения аудиоконтента из удаленных источников. Приложение ориентировано на минимализм, высокую скорость работы, полностью бесплатный доступ отсутствие ограничений на контент и гибкость управления через консольные команды.**

**Ключевой функционал**
* Сетевая архитектура: Программа использует модель динамической загрузки данных. Список композиций не вшит в код, а подтягивается в формате JSON с удаленного сервера, что позволяет обновлять библиотеку треков без перевыпуска клиентского приложения.

* Гибридная система кэширования: Реализован механизм интеллектуального сохранения данных. При первом прослушивании аудиопоток кэшируется в директорию временных файлов (Temp), что исключает повторную нагрузку на сеть при последующих вызовах того же трека.

* Интерфейс управления командами: В поле поиска интегрирован интерпретатор команд. Система распознает ключевые инструкции (Help, Reload, ClearCache) и выполняет их по нажатию клавиши Enter.

* Расширяемость (Custom JSON): Архитектура плеера поддерживает подключение пользовательских баз данных. Пользователь может переключить источник метаданных на собственный URL-адрес.

* Безопасность и отказоустойчивость
Валидация источников: При попытке смены стандартного адреса JSON система запрашивает подтверждение операции и выводит дисклеймер об ограничении ответственности разработчика за содержимое сторонних ресурсов.

* Визуальная индикация: В случае использования внешнего (стороннего) конфигурационного файла интерфейс активирует предупреждающий индикатор красного цвета.

* Автоматический откат (Rollback): При обнаружении ошибок в структуре внешнего файла или недоступности удаленного сервера программа автоматически выполняет сброс настроек до эталонного (стандартного) состояния, гарантируя непрерывную работу приложения.

**Перечень команд (вводятся в поиск)**
* help — Справочное руководство. Выводит в консоль список всех доступных команд и краткое описание их функций.

* reload — Обновление данных. Принудительно перезагружает текущий JSON-файл с сервера. Полезно, если автор добавил новые треки в плейлист.

* clearcache — Очистка памяти. Останавливает плеер и полностью удаляет все загруженные .mp3 файлы из временной папки (Temp). Освобождает место на диске.

* bsbconsole — Режим отладки. Переключает видимость окна консоли (справа). Позволяет видеть процесс загрузки, ошибки сервера и технические логи.

* changejson [url] — Смена источника. Позволяет подключить сторонний плейлист по прямой ссылке. Требует подтверждения безопасности во всплывающем окне.

**Инструкция по созданию и подключению плейлиста (JSON)**
* Чтобы создать собственный список воспроизведения, который будет корректно распознан плеером, необходимо выполнить три шага: подготовка данных, размещение в сети и активация в приложении.

**Шаг 1: Подготовка JSON-структуры**
* Создайте текстовый файл (например, my_music.json) и заполните его по следующему шаблону. Плеер ожидает массив объектов, где каждый объект — это одна песня.

* JSON
[
  {
    "name": "Название трека",
    "author": "Исполнитель",
    "url": "https://example.com/files/track1.mp3"
  },
  {
    "name": "Название трека",
    "author": "Исполнитель",
    "url": "https://example.com/files/track2.mp3"
  }
  #и т.д
]
* Технические требования к полям:

* name: Отображаемое название песни в списке.

* author: Имя автора.

* url: Прямая ссылка на файл в формате .mp3. Ссылка должна вести непосредственно на аудиопоток, а не на страницу проигрывателя (например, YouTube или SoundCloud не поддерживаются напрямую).

**Шаг 2: Размещение файла в сети**
* Для того чтобы плеер смог считать ваш список, файл должен быть доступен по прямой публичной ссылке.

* Рекомендуемый способ (GitHub): Залейте файл в свой репозиторий, откройте его и нажмите кнопку "Raw". Полученная ссылка в адресной строке будет идеальной для работы плеера.

* Альтернативы: Использование любых облачных хостингов, поддерживающих прямые HTTP/HTTPS запросы (например, прямые ссылки из Dropbox или личных серверов).

**Шаг 3: Подключение в BSB Music**
* После того как ссылка на ваш JSON-файл готова, выполните следующие действия в приложении:

* Активируйте поле поиска (центральная верхняя панель).

* Введите команду: changejson [ВАША_ССЫЛКА]

* ример: changejson https://raw.githubusercontent.com/user/music/main/list.json

* Нажмите клавишу Enter.

* В диалоговом окне подтвердите операцию, выбрав вариант "Да".

**Особенности работы системы:**
* Автоматическая проверка: Если структура файла нарушена или ссылка недоступна, плеер уведомит об ошибке и выполнит откат к стандартному плейлисту.

* Индикация: При успешной загрузке стороннего файла в интерфейсе появится метка красного цвета: * Внимание! У вас введен чужой JSON *.

* Сброс: Для возврата к стандартному списку достаточно перезапустить приложение или ввести команду reload.
